Comprehensive Technical Audit: Vendor-Desk

Main Idea:
The "Vendor-Desk" application is a mobile-first platform, developed using React Native with Expo, designed to facilitate operations for vendors. It integrates with a Supabase backend for data persistence, authentication, and potentially real-time functionalities. The codebase also includes a local API server component, which may provide additional business logic or serve as a proxy for backend interactions.

Technology Stack:
*   Frontend (Mobile): React Native with Expo, developed primarily in TypeScript. It leverages the Expo framework for simplified development, build processes, and cross-platform compatibility.
*   Backend/Database: Supabase is utilized as the primary backend-as-a-service, offering database capabilities (likely PostgreSQL), authentication, and real-time subscriptions.
*   API Communication: The application uses `@tanstack/react-query` for efficient data fetching, caching, and state management. The API client code (`lib/api-client-react`) is automatically generated using `orval` from an OpenAPI specification (`lib/api-spec`), ensuring type safety and consistency with the backend API.
*   Schema Validation: Zod is employed for defining and validating API schemas (`lib/api-zod`), providing robust data validation at runtime.
*   Package Management: The project is structured as a monorepo, managed by `pnpm`, which optimizes disk space usage and installation times for packages.
*   Local API Server: A Node.js-based API server (`artifacts/api-server`) is part of the project, suggesting custom backend logic, data processing, or serving as an intermediary for external services.

    *   `api-zod`: Zod schemas for validating data structures, ensuring type integrity between frontend and backend.

*   `artifacts/mobile/`: The core Expo React Native application.
